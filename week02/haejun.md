### 처음 접근 방식

오랜만에 시작하는 알고리즘 공부여서, 몸풀기겸으로 level 1 문제를 푼 것이라 아래의 풀이 방법과 동일하다.

### 풀이 방법

1. 결과를 저장할 빈 리스트 answer를 만든다.
2. 배열의 첫 번째 값을 forward에 저장하고 answer에 추가한다.
3. cursor를 1로 설정하여 배열의 두 번째 원소부터 탐색한다.
4. 현재 원소를 behind에 저장한다.
5. 이전 원소인 forward와 현재 원소인 behind를 비교한다.
6. 두 값이 다르면 연속된 중복 숫자가 아니므로 behind를 answer에 추가한다.
7. 비교가 끝난 후 forward를 현재 값인 behind로 변경한다.
8. 배열의 모든 원소를 확인할 때까지 위 과정을 반복한다.
9. 완성된 answer를 반환한다.

## 문제 2

### 문제 정보

- 문제명: 기능개발
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/42586
- 사용한 알고리즘 / 자료구조: 큐, 선형 탐색, 배열(List)

### 처음 접근 방식

각 기능이 완성되기까지 필요한 날짜를 먼저 계산한 뒤, 앞에 있는 기능의 배포일을 기준으로 함께 배포할 수 있는 기능의 개수를 세는 방식을 떠올렸다.

뒤의 기능이 먼저 완성되더라도 앞의 기능이 완료되지 않으면 배포할 수 없으므로, 단순히 완료 날짜가 같은 기능끼리 묶는 것이 아니라 현재 배포 기준일보다 빠르거나 같은 기능들을 연속해서 묶어야 한다고 판단했다.

각 기능의 남은 개발 일수를 계산해 complete_days에 저장하고, 앞에서부터 하나씩 꺼내면서 배포 묶음을 만들었다.

### 풀이 방법

1. progresses와 speeds의 첫 번째 원소를 각각 꺼낸다.
2. 현재 기능이 완성되기까지 필요한 날짜를 계산한다.
3. 필요한 날짜는 소수점이 발생할 수 있으므로 math.ceil()을 사용해 올림 처리한다.
4. 계산한 완료 일수를 complete_days 리스트에 저장한다.
5. 모든 기능의 완료 일수를 구한 뒤, complete_days의 첫 번째 값을 현재 배포 기준일인 deploy_day로 설정한다.
6. 현재 배포되는 기능의 수를 의미하는 deploy_amount를 1로 초기화한다.
7. 다음 기능의 완료 일수가 deploy_day보다 작거나 같으면 현재 기능과 함께 배포할 수 있으므로 제거하고 개수를 증가시킨다.
8. 다음 기능의 완료 일수가 deploy_day보다 크면 새로운 배포 묶음이므로 반복을 종료한다.
9. 계산한 배포 개수를 answer에 추가한다.
10. 모든 기능을 처리할 때까지 반복한 뒤 answer를 반환한다.

## 문제 3

### 문제 정보

- 문제명: 올바른 괄호
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/12909
- 사용한 알고리즘 / 자료구조: 스택, 문자열 순회

### 처음 접근 방식

올바른 괄호 문자열이 되려면 닫는 괄호가 나왔을 때 반드시 이전에 대응되는 여는 괄호가 존재해야 한다. 따라서 닫는 괄호를 만났는데 스택이 비어 있다면 즉시 올바르지 않은 괄호라고 판단할 수 있다.

문자열을 모두 확인한 뒤에도 스택에 여는 괄호가 남아 있다면 닫히지 않은 괄호가 존재하므로 False를 반환하도록 구현했다.

### 풀이 방법

1. 여는 괄호를 저장할 빈 리스트 brackets를 만든다.
2. 문자열 s를 앞에서부터 한 글자씩 확인한다.
3. 현재 문자가 (라면 brackets에 추가한다.
4. 현재 문자가 )라면 다음 두 가지 경우를 확인한다.
5. 스택이 비어 있다면 대응되는 여는 괄호가 없으므로 즉시 False를 반환한다.
6. 스택이 비어 있지 않다면 마지막 여는 괄호를 제거한다.
7. 문자열을 모두 확인한 뒤 스택이 비어 있으면 모든 괄호가 올바르게 짝지어진 것이므로 True를 반환한다.
8. 스택에 여는 괄호가 남아 있다면 닫히지 않은 괄호가 있다는 뜻이므로 False를 반환한다.

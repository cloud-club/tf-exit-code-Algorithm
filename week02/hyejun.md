## 문제 1

### 문제 정보

- 문제명: 올바른 괄호
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/12909
- 사용한 알고리즘 / 자료구조:

### 처음 접근 방식

여는 괄호와 닫는 괄호의 개수를 각각 세어서 두 개수가 같으면 올바른 괄호라고 판단하려고 했다.

### 풀이 방법

문자열을 앞에서부터 순회하면서 여는 괄호가 나오면 count를 1 증가시키고, 닫는 괄호가 나오면 1 감소시켰다. 순회 중 count가 음수가 되면 여는 괄호보다 닫는 괄호가 먼저 나온 것이므로 바로 False를 반환했다. 문자열을 모두 확인한 뒤 count가 0이면 모든 괄호가 짝지어진 것이므로 True를 반환했다.

### 어려웠던 지점

여는 괄호와 닫는 괄호의 전체 개수만 같다고 해서 올바른 괄호가 되는 것은 아니었다.

### 해결 방법

문자열을 순서대로 확인하면서 count가 중간에 음수가 되는 경우를 따로 검사했다.

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n)
- 공간복잡도: O(1)

### 새롭게 알게 된 점

괄호 문제에서는 전체 개수뿐만 아니라 괄호가 등장하는 순서도 확인해야 한다는 것을 새로 알게 되었다. 여는 괄호의 개수를 누적해서 관리하면 스택 없이도 해결할 수 있다.

## 문제 2

### 문제 정보

- 문제명: 최댓값과 최솟값
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/12939
- 사용한 알고리즘 / 자료구조:

### 처음 접근 방식

공백을 기준으로 문자열을 나눈 뒤, 숫자들을 직접 비교하면서 최댓값과 최솟값을 찾으려고 했다.

### 풀이 방법

문자열을 split()으로 나눈 뒤 각 값을 정수로 변환했다. 변환한 숫자 리스트에서 min(), max()를 사용해 최솟값과 최댓값을 구하고 문자열 형태로 합쳐 반환했다.

### 어려웠던 지점

문자열 상태에서 최댓값과 최솟값을 비교하면 숫자의 크기가 아니라 문자열 순서로 비교될 수 있다는 점을 주의해야 했다.

### 해결 방법

공백으로 나눈 값을 먼저 정수형으로 변환한 뒤 비교했다.

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n)
- 공간복잡도: O(n)

### 새롭게 알게 된 점

문자열로 입력된 숫자를 비교할 때에는 반드시 정수형으로 변환해야 한다. 이를 map(int, s.split())을 사용하여 간단하게 처리할 수 있다.

## 문제 3

### 문제 정보

- 문제명: 숫자의 표현
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/12924
- 사용한 알고리즘 / 자료구조: 완전탐색, 반복문

### 처음 접근 방식

각 자연수를 시작점으로 잡고, 그 수부터 연속된 숫자를 계속 더하며 합이 n이 되는 경우를 찾으려고 했다.

### 풀이 방법

0부터 n-1까지 반복하면서 각 위치를 연속된 자연수의 시작점으로 사용했다. 시작점부터 숫자를 하나씩 더해 합을 구하고, 합이 n과 같아지면 count를 증가시켰다. 합이 n보다 커지면 더 이상 확인할 필요가 없기에, 반복을 종료했다.

### 어려웠던 지점

반복문 안에서 시작점으로 사용한 변수 i를 계속 증가시켜도 되는지 헷갈렸다. 또한 합이 n을 넘은 뒤에도 반복하면 불필요한 연산이 많아질 수 있을 것이라는 우려가 생겼다.

### 해결 방법

각 시작점마다 별도의 변수를 사용해서 연속된 수를 증가시키는 방식으로 정리했다. 합이 n보다 커지면 바로 반복을 종료하도록 수정할 수 있다.

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(n²)
- 공간복잡도: O(1)

### 새롭게 알게 된 점

연속된 숫자의 합을 구하는 문제는 시작점을 하나씩 정해 완전탐색으로 풀 수 있다. 만약 입력 크기가 더 크다면 투 포인터 방식으로 개선할 수 있다고 한다.

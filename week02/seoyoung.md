## 문제 1

### 문제 정보

- 문제명: 프로세스
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/42587
- 사용한 알고리즘 / 자료구조: Queue, 시뮬레이션

### 처음 접근 방식

큐의 가장 앞에 있는 프로세스를 하나씩 꺼내면서 현재 프로세스보다 우선순위가 높은 작업이 남아 있는지 확인한다.

### 풀이 방법

1. `(우선순위, 원래 인덱스)` 형태로 큐를 생성한다.
2. 큐의 가장 앞 프로세스를 꺼낸다.
3. 남아 있는 큐에 더 높은 우선순위가 있는지 확인한다.
4. 있다면 현재 프로세스를 다시 큐의 뒤에 넣는다.
5. 없다면 실행 횟수를 증가시킨다.
6. 실행한 프로세스의 인덱스가 `location`이면 실행 순서를 반환한다.

### 어려웠던 지점

프로세스가 계속 큐 뒤로 이동하기 때문에 처음 위치를 어떻게 따라갈지 생각이 필요했다. 

### 해결 방법

우선순위만 저장하지 않고 `(우선순위, 인덱스)` 형태로 저장하여 원래 위치를 계속 추적했다.

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(N²)
- 공간복잡도: O(N)

### 새롭게 알게 된 점

큐에서 원소의 위치가 계속 변하는 경우에는 원래 인덱스를 함께 저장하면 원하는 원소를 쉽게 추적할 수 있다.

## 문제 2

### 문제 정보

- 문제명: 주식가격
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/42584
- 사용한 알고리즘 / 자료구조: Stack(모노토닉 스택)

### 처음 접근 방식

각 가격마다 뒤의 가격을 모두 확인하면서 언제 가격이 떨어지는지 찾는다.

### 풀이 방법

1. 가격이 떨어지지 않은 인덱스를 스택에 저장한다.
2. 현재 가격이 스택의 마지막 가격보다 작다면 스택에서 인덱스를 꺼낸다.
3. 현재 인덱스와 이전 인덱스의 차이를 정답에 저장한다.
4. 현재 인덱스를 스택에 추가한다.
5. 반복문이 끝난 뒤에도 스택에 남은 인덱스는 마지막 시점까지 가격이 유지된 경우이므로 마지막 인덱스와의 차이를 저장한다.

### 어려웠던 지점

인덱스를 저장할지 고민했다. 

### 해결 방법

가격이 유지된 시간을 계산하려면 인덱스 차이가 필요하므로 스택에는 가격이 아니라 인덱스를 저장했다.

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(N)
- 공간복잡도: O(N)

### 새롭게 알게 된 점

중첩 반복문처럼 보여도 각 원소가 스택에 한 번 들어가고 한 번 나오기만 하면 전체 시간복잡도는 O(N)이 된다.

## 문제 3

### 문제 정보

- 문제명: 전화번호 목록
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/42577
- 사용한 알고리즘 / 자료구조: 정렬, 문자열

### 처음 접근 방식

모든 전화번호를 하나씩 비교하여 접두어 관계를 찾으려고 했다.

### 풀이 방법

1. 전화번호를 문자열 기준으로 정렬한다.
2. 정렬된 배열에서 현재 번호와 다음 번호만 비교한다.
3. 다음 번호가 현재 번호로 시작하면 False를 반환한다.
4. 끝까지 접두어가 발견되지 않으면 True를 반환한다.

### 어려웠던 지점

왜 정렬만 하면 모든 쌍을 비교하지 않아도 되는지 이해하는 것이 어려웠다.

### 해결 방법

문자열을 정렬하면 서로 관계가 있는 문자열은 항상 서로 인접하게 배치된다.
119
1195524421
97674223
이런 식이라, 이렇게 정렬된다면 바로 다음 문자열만 비교하면 된다. 

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(N log N)
- 공간복잡도: O(N)

### 새롭게 알게 된 점

문자열의 접두어는 `startswith()`를 사용하면 쉽게 확인할 수 있다.

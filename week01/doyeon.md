# 오답노트
## 문제 1

### 문제 정보

- 문제명: 문자열 내 p와 y의 개수
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/12916
- 사용한 알고리즘 / 자료구조: 문자열 순회, 조건문, 카운팅

### 처음 접근 방식

문자열을 한 글자씩 확인하면서 `p` 또는 `P`가 등장하면 변수 `a`를 증가시키고, `y` 또는 `Y`가 등장하면 변수 `b`를 증가시켜 문자열 전체를 순회한 뒤 두 변수의 값이 같은지 비교하여 결과를 반환하는 방식으로 접근했다. (파이썬 메서드에 대해 다 까먹음..)

### 풀이 방법
나의 풀이

1. `p`의 개수를 저장할 변수 `a`와 `y`의 개수를 저장할 변수 `b`를 0으로 초기화한다.
2. 문자열 `s`를 한 글자씩 순회한다.
3. 현재 문자가 `p` 또는 `P`이면 `a`를 1 증가시킨다.
4. 현재 문자가 `y` 또는 `Y`이면 `b`를 1 증가시킨다.
5. 문자열 순회가 끝난 후 `a`와 `b`가 같은지 비교한다.
6. 두 값이 다르면 `False`, 같으면 `True`를 반환한다.

```python
def solution(s):
    a = 0
    b = 0

    for char in s:
        if char == 'p' or char == 'P':
            a += 1
        elif char == 'y' or char == 'Y':
            b += 1

    if a != b:
        return False

    return True
```

모범 답안

- 파이썬의 문자열 메서드를 사용하면 다음과 같이 더 간단하게 작성할 수도 있다.

```python
def solution(s):
    s = s.lower()
    return s.count('p') == s.count('y')
```

### 어려웠던 지점

오랜만에 코드를 작성해보니 파이썬 문법을 다 까먹은 나머지 문자열 반복문과 증가 연산자를 잘못 작성했다...  
또한 파이썬의 여러 문자열 메서드들을 활용하지 못 했다. 

### 해결 방법

문자열 순회 시 range 사용 X
```python
for char in s:
```

증감 연산자 작성 방식
```python
b += 1
```

대소문자를 각각 비교하는 대신 문자열 전체를 `lower()`로 소문자 변환한 뒤, `count()`를 사용
```python
s.lower().count('p') 
```
### 시간복잡도 / 공간복잡도

- 시간복잡도: O(N)
- 공간복잡도: O(1)

문자열의 길이를 `N`이라고 할 때 문자열을 한 번 순회하므로 시간복잡도는 O(N)이다. 개수를 저장하는 변수 두 개만 사용하므로 추가 공간복잡도는 O(1)이다.

### 새롭게 알게 된 점

- `lower()`를 사용하면 대소문자를 구분하지 않고 비교할 수 있다.
- `count()`를 사용하면 특정 문자의 개수를 간단하게 셀 수 있다.

## 문제 2

### 문제 정보

- 문제명: 신규 아이디 추천
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/72410
- 사용한 알고리즘 / 자료구조: 문자열 처리, 구현, 리스트 순회

### 처음 접근 방식

문제에서 제시한 7단계를 순서대로 구현하면 된다고 생각했다.

먼저 `lower()`를 사용해 대문자를 소문자로 변환하고, 문자열을 한 글자씩 순회하면서 허용되는 문자만 새로운 문자열에 추가하는 방식으로 접근했다.

연속된 마침표는 현재 문자가 `.`이고, 새 문자열의 마지막 문자도 `.`인 경우 추가하지 않도록 처리했다. 이후 문자열 양 끝의 마침표 제거, 빈 문자열 처리, 길이 제한 등의 조건을 차례대로 적용했다.

### 풀이 방법

1. `lower()`를 사용해 모든 대문자를 소문자로 변환한다.
2. 문자열을 한 글자씩 순회한다.
3. 현재 문자가 영문자나 숫자이거나 `-`, `_`, `.` 중 하나라면 결과 문자열에 추가한다.
4. 현재 문자가 `.`이고 결과 문자열의 마지막 문자도 `.`이라면 연속된 마침표이므로 추가하지 않는다.
5. `strip(".")`을 사용해 문자열 양 끝에 있는 마침표를 제거한다.
6. 문자열이 비어 있다면 `"a"`를 대입한다.
7. 슬라이싱으로 문자열을 앞의 15자까지만 남기고, 다시 끝의 마침표를 제거한다.
8. 문자열 길이가 3보다 작다면 마지막 문자를 반복해서 추가한다.
9. 최종 문자열을 반환한다.
```python
def solution(new_id):
    # 1단계
    new_id = new_id.lower()
    
    # 2, 3단계
    answer = ""
    for char in new_id:
        if char.isalnum() or char in "-_.":
            if char == "." and answer and answer[-1] == ".":
                continue
            else: 
                answer += char
    
    # 4단계
    answer = answer.strip(".")
    
    # 5단계
    if answer == "":
        answer = "a"
    
    # 6단계
    answer = answer[:15].strip(".")
    
    #7단계
    while len(answer) < 3:
        answer += answer[-1]
        
    
    return answer
```


### 어려웠던 지점

파이썬을 오랜만에 사용해 문자열 관련 기본 함수와 문법이 바로 떠오르지 않았다.

까먹은 문법들 ... 

- 문자열 길이를 구하는 `len()`
- 대문자를 소문자로 변환하는 `lower()`
- 문자열 양 끝의 문자를 제거하는 `strip()`
- 알파벳 또는 숫자인지 확인하는 `isalnum()`
- 문자열 슬라이싱
- 문자열은 직접 수정되지 않기 때문에 함수의 반환값을 다시 저장해야 한다는 점

또한 연속된 마침표를 제거할 때 결과 문자열이 비어 있는 상태에서 `answer[-1]`을 조회하면 오류가 발생할 수 있다는 점도 주의해야 했다.

다른 사람의 풀이에서는 정규식을 사용해 코드를 짧게 작성했지만, 정규식 문법이 기억나지 않아 바로 적용하기 어려웠다.

### 해결 방법

정규식을 억지로 사용하기보다, 현재 이해하기 쉬운 반복문과 조건문을 이용해 각 단계를 직접 구현했다.

연속된 마침표는 다음 조건을 사용해 처리했다.

```python
if char == "." and answer and answer[-1] == ".":
    continue
```

여기서 `answer`를 먼저 검사해 빈 문자열일 때 `answer[-1]`에 접근하지 않도록 했다.

다른 사람의 정규식 풀이와 비교하면서 `re.sub()`를 사용하면 특정 문자 제거와 연속된 마침표 처리를 더 짧게 구현할 수 있다는 것도 확인했다. 
### 시간복잡도 / 공간복잡도

- 시간복잡도: O(N²)
- 공간복잡도: O(N)

현재 코드에서는 반복문 안에서 다음과 같이 문자열을 계속 이어 붙인다.

```python
answer += char
```

파이썬 문자열은 불변 객체이므로 문자열을 추가할 때마다 새로운 문자열이 만들어질 수 있다. 따라서 이론적으로는 최악의 경우 O(N²)의 시간이 걸릴 수 있다.

문자열 길이가 짧은 문제이므로 실제 통과에는 문제가 없지만, 입력이 큰 문제에서는 리스트에 문자를 추가한 후 `join()`으로 합치는 방법이 더 적절하다.

```python
result = []

for char in new_id:
    if char.isalnum() or char in "-_.":
        if char == "." and result and result[-1] == ".":
            continue
        result.append(char)

answer = "".join(result)
```

이 방식으로 개선하면 전체 시간복잡도를 O(N)으로 볼 수 있다.

### 새롭게 알게 된 점

파이썬의 문자열은 불변 객체이므로 `lower()`, `strip()`, `replace()` 등의 함수를 호출해도 기존 문자열이 직접 변경되지 않는다는 점을 다시 확인했다.

```python
answer.strip(".")
```

위와 같이 호출만 하면 `answer`는 변경되지 않으며, 다음처럼 반환값을 다시 저장해야 한다.

```python
answer = answer.strip(".")
```

또한 다음 문자열 관련 문법을 다시 익혔다.

- `s.lower()`: 영문자를 소문자로 변환
- `s.isalnum()`: 문자열이 알파벳이나 숫자로 구성됐는지 확인
- `s.strip(".")`: 문자열 양 끝의 마침표 제거
- `len(s)`: 문자열 길이 확인
- `s[:15]`: 문자열의 앞 15자 추출
- `s[-1]`: 문자열의 마지막 문자 확인
- `"".join(list)`: 리스트에 저장된 문자들을 문자열로 결합

다른 사람의 풀이를 통해 정규식의 `re.sub()`도 다시 확인했다.

```python
re.sub('[^a-z0-9\-_.]', '', st)
```

허용되지 않는 문자를 제거하는 표현이다.

```python
re.sub('\.+', '.', st)
```

하나 이상 연속된 마침표를 하나의 마침표로 변경하는 표현이다.

다음에 비슷한 문자열 구현 문제를 풀 때는 먼저 필요한 문자열 함수와 슬라이싱 문법을 확인하고, 반복문 안에서 문자열을 계속 더하기보다 리스트와 `join()`을 활용하는 것을 기억해야겠다.

## 문제 3

### 문제 정보

- 문제명: 문자열 압축
- 문제 링크: https://school.programmers.co.kr/learn/courses/30/lessons/60057
- 사용한 알고리즘 / 자료구조: 문자열 슬라이싱, 완전탐색, 연속 구간 카운팅, 리스트

### 처음 접근 방식

문자열을 길이 `i`만큼 잘라서 배열로 만들고, 앞 조각과 현재 조각을 비교하면서 연속 개수를 세는 방식으로 접근했다.

처음에는 가장 많이 반복되는 횟수를 찾는 쪽으로 생각했지만, 이 문제는 압축된 문자열의 최소 길이를 구하는 문제였다. 그래서 단순히 최대 반복 횟수만 저장하는 것이 아니라, 각 길이별로 압축 결과를 만들어 길이를 비교해야 했다.

### 풀이 방법

`size`를 1부터 문자열 길이의 절반까지 늘려가며 모든 경우를 확인했다.

```python
def solution(s):
    if len(s) == 1:
        return 1

    answer = len(s)

    for size in range(1, len(s) // 2 + 1):
        compressed = ""
        prev = s[:size]
        count = 1

        for i in range(size, len(s), size):
            current = s[i:i + size]

            if current == prev:
                count += 1
            else:
                if count > 1:
                    compressed += str(count)
                compressed += prev
                prev = current
                count = 1

        if count > 1:
            compressed += str(count)
        compressed += prev

        answer = min(answer, len(compressed))

    return answer
```

예를 들어 `s = "aabbaccc"`이고 `size = 1`이면 다음처럼 잘린다.

```text
a / a / b / b / a / c / c / c
```

이때 연속된 같은 조각은 개수를 세어서 `2a2ba3c`처럼 압축된다.  
`size = 2`이면 다음처럼 잘린다.

```text
aa / bb / ac / cc
```

반복되는 구간이 적으면 그대로 두고, 마지막 조각은 반복문이 끝난 뒤 따로 붙여서 길이를 계산했다.

### 어려웠던 지점

처음에는 다음과 같은 부분에서 어려움을 겪었다.

- 문자열을 `size` 단위로 잘라 리스트를 만드는 방법
- `len()`과 슬라이싱 범위를 잘 맞추는 방법
- `current`와 `prev`를 비교하며 카운트를 유지하는 방법
- 마지막 묶음을 반복문 밖에서 따로 처리해야 한다는 점
- 반복 횟수가 10 이상일 때 자릿수까지 포함해서 길이를 계산해야 한다는 점

특히 `max` 값을 저장하는 방식으로 풀면 안 되고, 각 압축 단위마다 실제 압축 문자열의 길이를 만들어서 비교해야 한다는 점을 다시 정리해야 했다.

### 해결 방법

문자열을 길이 `i`만큼 자르는 부분은 리스트 컴프리헨션과 슬라이싱으로 처리했다.

```python
new_s = [s[j:j + i] for j in range(0, len(s), i)]
```

그다음에는 연속된 조각을 비교하면서 같은 값이면 `count`를 올리고, 값이 달라지는 순간 지금까지의 조각과 개수를 `compressed`에 붙였다.

마지막 조각은 반복문이 끝난 뒤 한 번 더 처리해야 누락이 없었다.  
반복 횟수가 1보다 크면 숫자를 앞에 붙이고, 1이면 조각만 붙이는 방식으로 압축 문자열의 길이를 계산했다.

다른 사람의 풀이를 보면서 `zip(words, words[1:] + [''])`처럼 마지막에 빈 문자열을 하나 붙여서 마지막 묶음까지 같은 루프로 처리할 수 있다는 점도 확인했다.  

### 시간복잡도 / 공간복잡도

- 시간복잡도: O(N²)
- 공간복잡도: O(N)

압축 단위를 1부터 문자열 길이의 절반까지 시도했다.

각 단위마다 문자열을 한 번씩 훑고 조각 리스트를 만들기 때문에 O(N)이 들고, 이를 여러 단위에 대해 반복하므로 전체 시간복잡도는 O(N²)이다.  
조각을 저장하는 리스트와 압축 문자열을 만들기 위한 문자열이 필요하므로 공간복잡도는 O(N)이다.

### 새롭게 알게 된 점

문자열 압축 문제는 가장 많이 반복된 패턴을 찾는 문제가 아니라, 모든 압축 단위를 시도해서 최단 길이를 찾는 완전탐색 문제라는 점을 다시 확인했다.

```python
for size in range(1, len(s) // 2 + 1):
```

또한 연속된 문자열은 현재 값과 이전 값을 비교하면서 카운트하는 방식이 가장 단순했다.

```python
if current == prev:
    count += 1
```

마지막 묶음은 반복문이 끝난 뒤 따로 붙여야 했고, 반복 횟수가 10 이상일 때는 `str(count)`의 길이까지 반영해야 했다.

```python
if count > 1:
    compressed += str(count)
```

다른 사람의 정답 코드에서 확인한 것처럼, 압축 결과 전체를 직접 만들지 않고 `len(word) + len(str(cnt))` 형태로 각 묶음의 길이만 더하는 방식도 깔끔했다.  
특히 `cnt > 1`일 때만 숫자 자릿수를 더하면 되므로, 압축 문자열을 실제로 조립하지 않아도 정답 길이를 구할 수 있었다.

다음에 비슷한 문제를 풀 때는 다음 내용을 기억해야겠다.

- 가능한 모든 단위를 시도하는 완전탐색 문제인지 확인하기
- 일정한 크기로 문자열이나 배열을 나눌 때 슬라이싱 사용하기
- 연속된 같은 값은 현재 값과 다음 값을 비교하며 카운팅하기
- 반복문 종료 후 마지막 값이 누락되지 않는지 확인하기
- 결과 문자열 전체가 필요하지 않다면 길이만 계산할 수 있는지 생각하기
